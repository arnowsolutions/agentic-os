🔴 P0 — Critical 3

P0-1

Zero Test Coverage

Severity: Criticalentire codebase88 .py + 73 .js files
No pytest, no Jest, no integration tests. Two scaffold test files with no assertions. Any change — even a simple refactor — risks production regression with no safety net.
Fix: Add 10 pytest smoke tests covering auth middleware, CRM CRUD, cost tracker calculations, agent routing, and selftest endpoint. ~2 hours. test_auth.py, test_cost_tracker.py, test_crm.py
P0-2

No Data Integrity Guarantees

Severity: Critical127 JSON filesNo transactions, no backups
All data lives in JSON files. Concurrent writes can corrupt data. No schema enforcement — any endpoint can write any shape. No foreign key enforcement — EZ IDs in eval forms may not exist in CRM. No automated backups. One corrupted write = potential data loss.
Fix: Add atomic writes (write to temp → rename) to CRM save. Set up daily backup cron job. ~1 hour.
P0-3

No Production Operations

Severity: CriticaldeploymentNo supervisor, monitoring, or alerting
Server runs via manual nohup python3 server.py. No systemd, no Docker restart policy, no process supervisor. Server crash = manual SSH restart. No staging environment — all changes deploy directly to production. No CI/CD.
Fix: Add systemd unit or Docker restart policy. Configure health check for Traefik. ~1 hour.

🟠 P1 — High Impact 4

No Mobile Support

styles.css · 70% of users are residents on phones

Zero @media queries. Sidebar consumes 70% of a 375px screen. No hamburger menu. Fixed font sizes. Tables overflow without scroll. The fix is ~50 lines of CSS.

No Type Safety

73 .js files · Zero TypeScript, JSDoc, or PropTypes

Renaming a field requires manual grep across 70 page files. IDE provides no autocomplete. A linter and JSDoc types would be high-ROI.

No Input Validation on Most Endpoints

server.py · Custom routes accept raw dicts

Routes like quick-action and PDF generation accept generic data: dict without Pydantic validation. Malformed input can cause 500 errors or data corruption.

No Rate Limiting on Login

modules/auth.py · 5-attempt lockout but no IP throttling

Brute-force across different accounts is possible. No per-IP rate limiting.

  • ✅ No-framework frontend — zero build step, fast iteration

Gaps

  • ❌ server.py still 3,400+ lines — ~60% of routes not yet extracted
  • ❌ No TypeScript — 73 JS files with zero type safety
  • ❌ No component reuse — each page duplicates header/table/filter patterns
  • ❌ JSON file database — no migrations, concurrency, or querying
  • ❌ No API versioning — all routes at /api/, no /api/v1/

Best Practices

Add spacing scale

Define 4-6 token values (xs/sm/md/lg/xl) and use consistently. Currently 8/10/12/14/16/20/24/32/40px used arbitrarily.

Add loading skeletons
